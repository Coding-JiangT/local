Git命令：
1、$ git init    初始化一个Git仓库
2、$ git add 文件名    添加文件
3、$ git commit -m "相应描述"    将添加的文件提交到仓库
4、$ git status     反映当前仓库中文件在工作区、暂存区的状态
5、$ git diff 文件名     查看文件的修改情况
6、$ git restore --staged 文件名      将文件从暂存区撤出，即撤销暂存区的修改，将文件状态恢复到add之前
   $ git restore --worktree 文件名    撤销在工作区而不在暂存区的文件的修改
7  $ git checkout -- file     让工作区的文件回到最近一次git commit或git add时的状态
   情况1：工作区文件修改后还未放到暂存区，则通过该命令可以将工作区文件恢复到与版本库一模一样的状态
   情况2：修改前文件被添加到了暂存区，则在工作区文件修改后，通过该命令会将文件恢复到与暂存区一模一样的状态
8、如果工作区文件误改后，添加到了暂存区，但没有提交，可通过 git reset HEAD <file>暂存区的修改撤销，
   再通过git checkout -- <file> 来丢弃工作区的修改
9、错误的版本提交到了版本库则需要用版本回退
10、$ git reset    版本回退
11、HEAD指向的版本就是当前版本
12、$ rm -rf .git  删除本地库
13、$ git rm   用于删除工作区的文件，删除后，如果想删除之前提交到版本库的该文件，则要再commit
